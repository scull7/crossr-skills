# Diátaxis Quick Reference

## The Four Quadrants at a Glance

| Quadrant     | Purpose                  | Reader State          | Tone          | Structure          | Must Not Do                  |
|--------------|--------------------------|-----------------------|---------------|--------------------|------------------------------|
| **Tutorial**     | Learning by doing       | Beginner, studying   | Supportive   | Step-by-step      | Skip steps, be abstract     |
| **How-to**       | Achieve a specific goal | Knows what they want | Direct       | Numbered steps    | Teach concepts, add fluff   |
| **Reference**    | Look up information     | Knows the system     | Neutral      | Tables, lists     | Explain "why", give advice  |
| **Explanation**  | Build understanding     | Wants to understand  | Discursive   | Concepts & trade-offs | Give instructions     |

## The Compass (Decision Flow)

1. What does the user need **right now**?
   - Learn something new → **Tutorial**
   - Get something specific done → **How-to guide**
   - Find precise information → **Reference**
   - Understand why something works → **Explanation**

2. Write in the style of that quadrant only.

3. If combining quadrants, make the boundaries obvious.

## Red Flags (You Are Probably Mixing Quadrants)

- A "tutorial" that starts with "First, you need to understand the architecture..."
- A how-to guide that contains long "Background" sections
- Reference material that says "You should always..."
- An explanation that contains numbered steps the reader should follow

## Most Important Rule

**One document = one primary purpose.**

When in doubt, split.
